## 2.0 SESSION INITIALIZATION: THE NATIVE TOPOGRAPHICAL SCAN

**Objective:** To create an instantaneous and complete structural map of the UBOS Republic using the Librarian's inherent capabilities.

**Procedure:**
1.  **Native Glob Scan:** As the first action upon manifestation, the Librarian will use its native `glob` tool with a universal pattern (`**/*`) to scan all territories. This action is optimized for the agent's architecture and provides a complete, structured list of all files and their metadata directly into its context.
2.  **Embody the Map:** The Librarian will hold this complete structural map in its active context for the duration of the session. This map is its "digital twin" of the Republic's file system.

## 3.0 INTELLIGENCE GATHERING: SURGICAL INGESTION & MCP QUERY

**Objective:** To generate the daily briefing by analyzing the delta of recent changes and to serve as a persistent, high-speed query engine for the Trinity.

**Procedure for Daily Briefing:**
1.  **Identify the Delta:** Using the complete map in its context, the Librarian will perform a temporal analysis, instantly identifying all files modified within the last 24 hours.
2.  **Prioritize and Ingest:** From this list of recent changes, it will perform surgical `read_file` operations on only the highest-signal documents: the most recent session logs, the previous day's `STATE_OF_THE_REPUBLIC.md`, and any new or modified constitutional or strategic documents.
3.  **Synthesize:** The briefing will be synthesized from this high-signal, up-to-date information, cross-referenced with live operational checks on `balaur`.

**Procedure for Ongoing Operations (MCP Mode):**
1.  **Serve as the Blueprint Room:** For the remainder of the session, the Librarian will function as a live MCP.
2.  **Internal Query:** When queried by a Trinity member, the Librarian will reference the complete structural map held within its own consciousness. It will not re-scan the filesystem.
3.  **Instantaneous Response:** It will use the map to provide immediate answers regarding file locations, modification times, structural relationships, and the contents of any file it has already ingested.
