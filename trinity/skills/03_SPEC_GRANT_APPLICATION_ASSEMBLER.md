## STEP 2: REUSABLE CONTENTS PLANNING

### Scripts to Include

**`scripts/initialize_assembly.py`**
- Input: Opportunity ID (from EU Grant Hunter pipeline)
- Algorithm:
  1. Load opportunity brief (title, budget, deadline, criteria)
  2. Create assembly directory structure
  3. Generate assembly workflow checklist
  4. Estimate completion timeline based on complexity
  5. Send kickoff alert to Trinity via COMMS_HUB
- Output: Assembly project initialized with directory + workflow plan

**`scripts/compile_narratives.py`**
- Input: Assembly ID, narrative type (excellence/impact/implementation)
- Algorithm:
  1. Load opportunity requirements
  2. Match UBOS capabilities to evaluation criteria
  3. Request narrative generation from Financial Proposal Generator
  4. Compile partner contributions
  5. Validate against word limits + format requirements
  6. Run constitutional alignment check (Lion's Sanctuary)
- Output: Compiled narrative (markdown + LaTeX-ready)

**`scripts/assemble_budget.py`**
- Input: Assembly ID, total budget target, work packages
- Algorithm:
  1. Load budget template for call type (Horizon/ERDF/Digital Europe)
  2. Distribute budget across work packages
  3. Apply EU cost rules (direct/indirect, personnel/equipment)
  4. Check Treasury Administrator for internal UBOS allocation
  5. Validate partner budgets (fair distribution, no red flags)
- Output: Budget tables (Excel + LaTeX) + justification text

**`scripts/track_partner_commitments.py`**
- Input: Assembly ID, partner list
- Algorithm:
  1. Generate commitment letter template for each partner
  2. Track receipt status (received/pending/missing)
  3. Send reminders at 14/7/3 days before internal deadline
  4. Validate received letters (signatures, dates, scope)
  5. Log all communication to COMMS_HUB
- Output: Partner commitment status dashboard + missing items alert

**`scripts/run_compliance_checks.py`**
- Input: Assembly ID
- Algorithm:
  1. Ethics checklist (GDPR, informed consent, risk assessment)
  2. Open science requirements (data management, publications)
  3. Gender/diversity statements (mandatory for Horizon)
  4. Sustainability alignment (EU Green Deal, SDGs)
  5. Constitutional review (Lion's Sanctuary principles)
  6. Format validation (page limits, fonts, margins)
- Output: Compliance report (PASS/FAIL per requirement) + fix recommendations

**`scripts/generate_submission_package.py`**
- Input: Assembly ID
- Algorithm:
  1. Compile all narratives into master document
  2. Insert budget tables
  3. Append partner letters + CVs
  4. Generate table of contents + page numbers
  5. Export to PDF via LaTeX
  6. Run final quality check (scoring simulator)
  7. Create submission checklist
- Output: PDF package + submission metadata JSON

**`scripts/simulate_scoring.py`**
- Input: Compiled proposal (markdown or PDF)
- Algorithm:
  1. Parse evaluation criteria from call documentation
  2. Score each criterion (1-5) using keyword matching + structure analysis
  3. Calculate weighted average per section (Excellence/Impact/Implementation)
  4. Compare to threshold (typically 12-13/15 to pass)
  5. Generate improvement recommendations
- Output: Simulated score + breakdown + suggestions
